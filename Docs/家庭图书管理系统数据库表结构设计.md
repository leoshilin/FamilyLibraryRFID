# 家庭图书管理系统 – 数据库表结构设计文档

## 1. 整体设计原则
### 1.1 一般原则
1. 家庭（Family）是系统的第一业务边界。

2. 系统级主数据（book_meta）与家庭级实体（book_item）严格分离

   * 系统级：书籍在“世界层面”的客观定义（跨家庭共享），定义书籍“是什么”（book_meta）

   * 家庭级：家庭“拥有什么书”（book_item），家庭中真实存在的一本实体书

3. 套装书是系统级出版形态，不是库存属性 （=套装书是系统级概念，不允许家庭私有定义）

4. RFID 为可选能力，允许为空、可重绑定

5. 所有库存变化都有记录（时间 + 理由 + 操作人）

6. book_meta 的数据除了封面图之外，一旦创建不允许修改

7. 对于可选字段，不使用 null，未赋值时直接省略该字段。

### 1.2 字段分类原则

系统字段分为三类：

  - Required Field（必须字段，标注为**必须**）
      例如：
      - _id
      - created_at
      - created_by
      - status

   - Optional Field（可选字段,标注为**可选**）
      仅在具有业务意义时存在。

      未赋值时：删除字段（Unset）。

      禁止使用：
      - null
      - ""
      - 默认占位值

      例如：

      - rfid_tid
      - currentFamilyId
      - updated_at
      - updated_by

   - Computed Field（系统计算字段，标注为**计算**）
      由系统自动维护。不得人工修改。

## 3. 表结构设计

### 3.1 user（用户主表）

| 字段名             | 类型          | 字段分类  | 中文说明                    |
| --------------- | ------------ | ------------ | ----------------------- |
| _id              | varchar(64)  |必须 | 系统用户ID，主键                 |
| openid            | varchar(64) |必须 | openid |
| nickName           | varchar(255) |必须| 名称                      |
| currentFamilyId         | varchar(64)      | **可选** | 当前登录家庭                    |
| role         | enum('ADMIN','USER')        | 必须 | 用户是系统管理员，或一般用户                    |
| status       | varchar(20) | 必须 | ACTIVE/DISABLED                     |
| created_at      | datetime  | 必须   | 创建时间                    |
| updated_at      | datetime    | **可选** | 更新时间                    |


**索引**
openid: 唯一索引



📌 **说明**：

* 用户和家庭是1：n的关系，因此currentFamilyId代表当前用户默认使用的家庭（最近一次选择的家庭）。系统中所有业务逻辑（图书检索等）都使用此数据。


### 3.2 family（家庭主表）

| 字段名             | 类型          | 字段分类  | 中文说明                    |
| --------------- | ------------ | ------------ | ----------------------- |
| _id              | varchar(64) | 必须 | 家庭ID，主键                 |
| name           | varchar(255) | 必须| 家庭名称                      |
| status       | varchar(20) | 必须| ACTIVE/DISABLED                     |
| created_by         | varchar(64)        | 必须 | created_by 仅记录创建人。真正Owner身份由user_family.role决定（可以被转移）。                    |
| created_at      | datetime    | 必须| 创建时间                    |
| updated_by         | varchar(64)        | **可选** | 家庭更新者                    |
| updated_at      | datetime    | **可选** | 更新时间                    |


📌 **说明**：

* created_by 代表的是该家庭由谁创建，并不是权限的含义。家庭的管理者，还是一般用户的角色分配在user_family表中定义。


### 3.3 user_family（用户家庭对应主表）

| 字段名             | 类型           | 字段分类 | 中文说明                    |
| --------------- | ------------ | ------------ | ----------------------- |

| userId              | varchar(64) | 必须 | 用户ID                 |

| role              |  enum('OWNER','MEMBER','GUEST')  | 必须 | family role                 |

**索引**


📌 **说明**：

* role 仅表示角色身份，不直接表示权限。实际权限由系统内置 rolePermission 配置决定。
* 业务约束： 同一用户，不能重复加入同一家庭

### 3.6 book_item（家庭书籍实体表）

| 字段名          | 类型          | 字段分类 | 中文说明                     |
| ------------ | ----------- | ------------| ------------------------ |
| _id           | varchar(64) | 必须 | 实体书 ID，主键                   |
| family_id    | varchar(64) | 必须 | 所属家庭                     |
| book_meta_id | varchar(64) | 必须 | 关联书籍主数据                  |
| bookshelf_id | varchar(64) | 必须 | 所在书架                     |
| set_index    | int         | **可选**| 套装内序号（仅套装书）                    |
| inventory_status       | enum        | 必须 | in_stock / off_stock |
| rfid_tid       | varchar(64)       | **可选**| RFID标签的tid（对应唯一的物理标签），仅绑定RFID时存在，未绑定时字段不存在 |
| fg_delete   | boolean | **可选**| 书本逻辑删除标记，无法再次重新上架                      |
| created_at   | datetime    | 必须 | 入库时间                     |
| updated_at   | datetime    | **可选**| 状态更新时间                     |

**索引**
rfid_tid: 唯一索引

📌 **说明**：
* fg_delete：下架图书的彻底删除（逻辑）标志。删除前下架图书仍然可以恢复上架，但删除后图书则无法恢复上架。
* 彻底删除前必须处于 off_stock 状态。 禁止：  in_stock -> fg_delete=true

| inventory_status    | fg_delete | 含义    |
| --------- | --------- | ----- |
| in_stock  | false     | 在架    |
| off_stock | false     | 下架（可重新上架）   |
| off_stock | true      | 已彻底删除 |
| in_stock  | true      | 非法状态  |

## 4. 关键索引建议

## 5. 未来功能扩展相关表结构（当前不实现）
### 5.1 user（用户表）

| 字段名        | 类型           | 中文说明                  |
| ---------- | ------------ | --------------------- |
| id         | varchar(64)  | 用户主键 ID（系统生成）         |
| openid     | varchar(64)  | 微信 openid（应用内唯一）      |
| unionid    | varchar(64)  | 微信 unionid（跨应用唯一，可为空） |
| nickname   | varchar(100) | 用户昵称                  |
| created_at | datetime     | 创建时间                  |

**唯一性规则说明：**

* openid 在同一小程序内唯一

* unionid 若存在则用于跨系统用户合并

* 系统内部统一使用 id 作为外键

**设计预留说明：**

* 若未来支持多登录来源，可引入 user_auth 表，将 openid / unionid 迁移至该表。
