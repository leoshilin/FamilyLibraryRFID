# 家庭图书管理系统 – 数据库表结构设计文档

## 1. 整体设计原则
### 1.1 一般原则
1. 家庭（Family）是系统的第一业务边界。

2. 系统级主数据（book_meta）与家庭级实体（book_item）严格分离

   * 系统级：书籍在“世界层面”的客观定义（跨家庭共享），定义书籍“是什么”（book_meta）

   * 家庭级：家庭“拥有什么书”（book_item），家庭中真实存在的一本实体书

3. 套装书是系统级出版形态，不是库存属性 （=套装书是系统级概念，不允许家庭私有定义）

4. RFID 为可选能力，允许为空、可重绑定

5. 所有库存变化都有记录（时间 + 理由 + 操作人）

6. book_meta 的数据除了封面图之外，一旦创建不允许修改

7. 对于可选字段，不使用 null，未赋值时直接省略该字段。

### 1.2 字段分类原则

系统字段分为三类：

  - Required Field（必须字段，标注为**必须**）
      例如：
      - _id
      - created_at
      - created_by
      - status

   - Optional Field（可选字段,标注为**可选**）
      仅在具有业务意义时存在。

      未赋值时：删除字段（Unset）。

      禁止使用：
      - null
      - ""
      - 默认占位值

      例如：

      - rfid_tid
      - current_family_id
      - updated_at
      - updated_by

   - Computed Field（系统计算字段，标注为**计算**）
      由系统自动维护。不得人工修改。

### 3.6 book_item（家庭书籍实体表）

| 字段名          | 类型          | 字段分类 | 中文说明                     |
| ------------ | ----------- | ------------| ------------------------ |
| _id           | varchar(64) | 必须 | 实体书 ID，主键                   |
| family_id    | varchar(64) | 必须 | 所属家庭                     |
| book_meta_id | varchar(64) | 必须 | 关联书籍主数据                  |
| bookshelf_id | varchar(64) | 必须 | 所在书架                     |
| set_index    | int         | **可选**| 套装内序号（仅套装书）                    |
| inventory_status       | enum        | 必须 | in_stock / off_stock |
| rfid_tid       | varchar(64)       | **可选**| RFID标签的tid（对应唯一的物理标签），仅绑定RFID时存在，未绑定时字段不存在 |
| fg_delete   | boolean | **可选**| 书本逻辑删除标记，无法再次重新上架                      |
| created_at   | datetime    | 必须 | 记录创建时间（实体书数据记录创建时刻，仅初次上架时写入一次，重新上架不再刷新）                     |
| on_shelf_at   | datetime    | 必须 | 上架时间（初次上架与重新上架时写入/刷新；表示实体书当前"在架状态"的起始时刻，用于"最近上架"排序与按上架时间筛选）                     |
| updated_at   | datetime    | **可选**| 记录更新时间（任何写操作——下架 / 重新上架 / 更换书架 / 彻底删除 / 绑定解绑RFID——均刷新）                     |

**索引**
rfid_tid: 唯一索引

📌 **说明**：
* fg_delete：下架图书的彻底删除（逻辑）标志。删除前下架图书仍然可以恢复上架，但删除后图书则无法恢复上架。
* 彻底删除前必须处于 off_stock 状态。 禁止：  in_stock -> fg_delete=true
* **三时间字段语义区分**（book_item）：
  | 字段 | 语义 | 写入时机 |
  | ---- | ---- | ---- |
  | `created_at` | 实体书**数据记录**创建时刻 | 仅初次上架（`api_bookitem_create`）写入一次，重新上架不刷新 |
  | `on_shelf_at` | 当前**在架状态**的起始时刻（上架时间） | 初次上架写入；重新上架（`api_bookitem_restock`）刷新；下架 / 更换书架 / 彻底删除 / 绑定解绑RFID 不刷新 |
  | `updated_at` | 记录**末次更新**时刻 | 任意写操作（下架、重新上架、更换书架、彻底删除、绑定/解绑RFID）均刷新 |
  > 注：`api_book_search` 的"上架期间"筛选与 `api_book_searchRecent` 的"最近上架"排序，以及详情页"上架日"展示，均基于 `on_shelf_at`，而非 `created_at`，以避免重新上架导致"最近上架"列表重排（见 API 文档"三、遗留问题"第 5 条）。
* book_item 始终以 `in_stock` 状态创建，故 `on_shelf_at` 在创建时必有值，归为"必须"字段。

| inventory_status    | fg_delete | 含义    |
| --------- | --------- | ----- |
| in_stock  | false     | 在架    |
| off_stock | false     | 下架（可重新上架）   |
| off_stock | true      | 已彻底删除 |
| in_stock  | true      | 非法状态  |

> **任务队列精简原则（重要）**：`device_task` 仅承担调度职责，只保存 `book_item_id` / `target_tid` 等必要的调度字段，**不冗余存储 ISBN / 书名 / 作者等展示字段**。
> - 展示字段（ISBN / 书名 / 作者）由 `api_task_accept`（J1）在 PDA 领取任务时，经 `book_item.book_meta_id → book_meta` 实时关联拼装后随任务返回，PDA 直接显示并用于 ISBN 校验。
> - 不在创建任务（G1 `api_task_createBindRfid` / G2 `api_task_createFindBook`）时反查写入，避免任务表与 book_meta 出现数据冗余与不一致。
> - 历史上曾规划 `book_title` / `book_author` / `book_isbn` 三个展示字段入任务表，后废弃移除（见功能需求与流程定义书 F4.3 / F6.2 修订说明）。

#### 3.9.1 `result` 结构约定（执行结果 JSON）

`result` 为可选 JSON，由 PDA 经 `api_task_complete`（J2）在任务结束时写入，结构随 `task_type` 与 `status` 不同而不同。以下为约定结构：

**find_book（寻书）：**
| 字段 | 类型 | 说明 | 出现条件 |
| ---- | ---- | ---- | ---- |
| `found` | boolean | 是否读到目标标签连续结果 | 始终 |
| `reason` | enum | 失败原因：`no_rfid_read`（未读到 RFID 即结束）/ `user_abort`（任务台主动放弃）/ `device_error`（PDA 设备异常） | `status = failed` |
| `durationMs` | int | 寻书会话耗时（毫秒） | 始终 |
| `foundRssi` | int | 结束时最后读到的 RSSI（未找到则为 null / 省略） | 找到时 |
| `readCount` | int | 目标标签连续稳定读取次数（判定「已找到」的计数） | 始终 |

> **关于 abort 操作**：用户通过 `api_task_abort`（J6）退出寻书时，任务状态从 `running` 回退为 `pending`（清空 `claimed_by_device` / `claimed_at`），**不写入 `result` 字段，不设置 `completed_at`**。此操作不产生任何持久化结果记录，任务干净回退到待领取状态。与 `failed` 不同，`abort` 后的任务可被再次轮询领取执行。

- `success` 示例：`{ "found": true, "durationMs": 12345, "foundRssi": -28, "readCount": 12 }`
- `failed` 示例：`{ "found": false, "reason": "no_rfid_read", "durationMs": 30210, "readCount": 0 }`

**bind_rfid（绑定）：**
| 字段 | 类型 | 说明 | 出现条件 |
| ---- | ---- | ---- | ---- |
| `epcWritten` | boolean | EPC 回写是否成功 | `status = success` |
| `reason` | enum | 失败原因：`user_abort` / `bind_failed` / `device_error` | `status = failed` |

> 约定：失败原因统一放在 `result.reason`；PDA 任务台「最近完成任务」列表（J5 `api_task_listRecentCompleted`）据此展示失败理由，小程序端未来排查失败任务亦复用此字段。

#### 3.9.2 执行时间 / 最近完成查询依赖

- `completed_at`：任务结束（`success` / `failed`）时由云端写入；PDA 任务台「最近完成任务」查询（J5）以 `status ∈ [success, failed]` 且 `completed_at` 在 **最近 3 天**内 为筛选条件，按 `completed_at` 倒序返回。
- 展示字段（ISBN / 书名 / 作者）不在 `device_task` 冗余存储，J5 与 J1 一致经 `book_item.book_meta_id → book_meta` 实时关联拼装后随任务返回。
- 续领（崩溃恢复）：J1 轮询时 `running` 任务仅允许**本设备已领取**（`claimed_by_device = 本设备`）的重新进入候选，避免抢占其它设备正在执行的任务。

## 4. 关键索引建议

* book_meta.isbn

* book_meta.is_set

* book_item.family_id + status

* book_item.family_id + on_shelf_at（支撑"最近上架"排序与按上架时间筛选）

* book_item.rfid_tid

* device_task.status + created_at（支撑 PDA 轮询领取：pending/running 按创建时间升序取候选）

* device_task.status + completed_at（支撑 PDA 任务台「最近完成任务」查询：success/failed 按完成时间倒序，近 3 天过滤）

* device_task.claimed_by_device + status（支撑续领：仅本设备已领取的 running 任务重新进入候选，避免抢占其它设备）
