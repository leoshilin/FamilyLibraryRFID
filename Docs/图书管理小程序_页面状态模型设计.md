# 图书管理小程序 ------ 页面状态模型设计文档

## 一、整体功能结构

    首页
     ├── 最近上架（最多5本）
     │     ├── 点击图书 → 图书详情页（detail模式）
     │     └── 点击“查看全部” → 在库列表页
     │             └── 点击图书 → 图书详情页（detail模式）
     │
     ├── 添加藏书（扫码）
     │     └── 图书确认页（confirm模式）
     │             └── 上架成功 → 返回首页并刷新
     │
     ├── 寻书
     │     └── 搜索结果列表页
     │             ├── 左滑/右滑 → 出库
     │             └── 点击图书 → 图书详情页（detail模式）
     │
     ├── 在库（入口预留）
     ├── 出库（入口预留）
     └── 设置（入口预留）

------------------------------------------------------------------------

# 二、页面状态模型设计

------------------------------------------------------------------------

## 1️⃣ 首页（Index Page）

### 状态模型

    [Loading]
        ↓
    [Loaded]
        ├── 无数据
        └── 有最近上架数据（最多5本）

### 触发事件

  事件           行为                     状态变化
  -------------- ------------------------ ------------------
  下拉刷新       重新拉取数据             Loading → Loaded
  点击图书       跳转详情页(detail模式)   不改变首页状态
  点击查看全部   跳转在库列表页           不改变首页状态
  上架成功返回   重新刷新数据             Loading → Loaded

------------------------------------------------------------------------

## 2️⃣ 图书确认/详情页（Book Page）

> 统一页面，通过 mode 控制

## 页面模式

    mode = confirm | detail

------------------------------------------------------------------------

## 状态模型

    [Loading]
       ↓
    [Loaded]
       ├── confirm模式
       │       ├── 可编辑套装信息
       │       └── 显示“上架”按钮
       │
       └── detail模式
               ├── 只读
               ├── 显示“下架”按钮
               └── 显示“绑定RFID”按钮（预留）

------------------------------------------------------------------------

## confirm 模式状态流

    初始
     ↓
    点击上架
     ↓
    [Submitting]
     ↓
    成功 → Toast → 返回首页 → 首页刷新
    失败 → 恢复按钮

------------------------------------------------------------------------

## detail 模式状态流

    正常查看
     ├── 点击下架 → 选择出库原因 → 成功 → 返回上一页 → 刷新列表
     └── 点击绑定RFID → 绑定成功 → 状态更新

------------------------------------------------------------------------

# 3️⃣ 在库列表页（全部已上架图书）

### 状态模型

    [Loading]
       ↓
    [Loaded]
       ├── 空列表
       └── 有数据列表

### 行为

  行为       结果
  ---------- --------------------
  点击图书   跳转详情页(detail)
  下拉刷新   重新加载
  滚动分页   加载更多

------------------------------------------------------------------------

# 4️⃣ 寻书页面

## 页面结构

    搜索框
      ↓
    搜索结果列表

------------------------------------------------------------------------

## 状态模型

    [Idle]
       ↓
    输入关键词
       ↓
    [Searching]
       ↓
    [Result]
       ├── 空结果
       └── 匹配结果列表

------------------------------------------------------------------------

## 搜索匹配规则

-   模糊匹配书名
-   模糊匹配作者
-   不区分大小写

------------------------------------------------------------------------

## 列表交互状态

    正常状态
      ├── 左滑 → 显示“出库”按钮
      ├── 右滑 → 显示“出库”按钮
      └── 点击 → 进入详情页(detail)

------------------------------------------------------------------------

## 出库流程

    滑动点击出库
       ↓
    选择原因（售卖 / 赠送 / 丢失 / 损坏 / 其他）
       ↓
    确认
       ↓
    数据库状态改为 out
       ↓
    列表自动移除该书

------------------------------------------------------------------------

# 三、全局数据状态模型

    Book Status:
      - in_stock
      - out_stock

------------------------------------------------------------------------

# 四、页面间刷新策略

  来源页面      返回后刷新方式
  ------------- --------------------
  confirm成功   调用首页刷新方法
  出库成功      调用来源列表页刷新
  绑定RFID      本页更新数据

------------------------------------------------------------------------

# 五、未来可扩展点

-   RFID唯一绑定校验
-   出库历史记录表
-   图书生命周期追踪
-   数据统计页面

------------------------------------------------------------------------

# 六、总结架构

    首页（展示 + 入口）
       ↓
    图书详情（核心管理页）
       ↓
    状态变更（入库 / 出库 / RFID）

这是一个"图书生命周期管理系统"的基础状态模型。
