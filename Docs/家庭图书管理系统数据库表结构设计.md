# 家庭图书管理系统 – 数据库表结构设计文档

## 一、整体设计原则

1. 家庭（Family）是系统的第一业务边界，但当前版本固定**family_id** 为**fm00001**。不设定Family的主数据表。

2. 系统级主数据（book_meta）与家庭级实体（book_item）严格分离

   * 系统级：书籍在“世界层面”的客观定义（跨家庭共享），定义书籍“是什么”（book_meta）

   * 家庭级：家庭“拥有什么书”（book_item），家庭中真实存在的一本实体书

3. 套装书是系统级出版形态，不是库存属性 （=套装书是系统级概念，不允许家庭私有定义）

4. RFID 为可选能力，允许为空、可重绑定

5. 所有库存变化都有记录（时间 + 理由 + 操作人）

6. book_meta 的数据除了封面图之外，一旦创建不允许修改

## 三、表结构设计（系统基础数据）

### 1. user（用户主表）

| 字段名             | 类型           | 中文说明                    |
| --------------- | ------------ | ----------------------- |
| _id              | varchar(64)  | 系统用户ID                 |
| openid            | varchar(64)  | openid |
| nickName           | varchar(255) | 名称                      |
| currentFamilyId         | varchar(64)         | 当前登录家庭                    |
| status       | varchar(20) | ACTIVE/INACTIVE                     |
| created_at      | datetime     | 创建时间                    |
| updated_at      | datetime     | 更新时间                    |


📌 **说明**：

* 用户和家庭是1：n的关系，因此currentFamilyId代表当前用户默认使用的家庭（最近一次选择的家庭）。系统中所有业务逻辑（图书检索等）都使用此数据。


### 2. family（家庭主表）

| 字段名             | 类型           | 中文说明                    |
| --------------- | ------------ | ----------------------- |
| _id              | varchar(64)  | 家庭ID                 |
| name           | varchar(255) | 家庭名称                      |
| status       | varchar(20) | Active/Inactive                     |
| created_by         | varchar(64)          | 家庭创建者                    |
| created_at      | datetime     | 创建时间                    |
| updated_by         | varchar(64)          | 家庭更新者                    |
| updated_at      | datetime     | 更新时间                    |


📌 **说明**：

* created_by 代表的是该家庭由谁创建，并不是权限的含义。家庭的管理者，还是一般用户的权限分配在user_family表中定义。


### 3. user_family（用户家庭对应主表）

| 字段名             | 类型           | 中文说明                    |
| --------------- | ------------ | ----------------------- |
| _id              | varchar(64)  | 用户家庭对应关系ID                 |
| userId              | varchar(64)  | 用户ID                 |
| familyId              | varchar(64)  | 家庭ID                 |
| role              |  enum('OWNER','MEMBER','GUEST')  | 用户的访问权限                 |
| joined_at      | datetime     | 加入家庭的时间                    |

📌 **说明**：

* 本系统对操作权限简化为四种权限定义。系统中所有功能所需的权限，在每个功能描述处定义。
  * A：admin权限（系统级别操作权限）
  * O：Owner权限（家庭级别操作权限）
  * M：Member权限（家庭级别部分操作权限）
  * G：Guest权限（家庭级别只读权限）

* role的分类如下，每个role对应各自的三种权限。
  * Owner：具备O的权限（家庭中所有的操作权限，例如可修改家庭/书架的名称，可绑定RFID，上下架图书等）
  * Member：具备M权限（例如可创建寻书任务）
  * Guest：具备G权限（仅可在手机端检索查看书架和图书）

* 业务约束
同一用户，不能重复加入同一家庭

## 三、表结构设计（系统业务数据）

### 1. book_meta（系统级书籍主数据表）

| 字段名             | 类型           | 中文说明                    |
| --------------- | ------------ | ----------------------- |
| _id              | varchar(64)  | 系统书籍 ID                 |
| isbn            | varchar(20)  | ISBN（唯一）                |
| title           | varchar(255) | 书名                      |
| authors         | varchar(255) / json         | 作者列表                    |
| publisher       | varchar(255) | 出版社                     |
| publish_year    | varchar(10)  | 出版年份                    |
| price           | varchar(50)  | 定价（原始字符串，包含了货币单位）               |
| binding         | enum         | 平装 / 精装 / 未知            |
| is_set          | boolean      | 是否套装书                   |
| set_total_count | int          | 套装总册数（未知可为空）                   |
| cover_url          | varchar(255) | 封面图片地址（云存储 URL）                  |
| source          | enum('none','douban','official（预留）') | 封面来源                  |
| cover_updated_at      | datetime                         | 封面更新时间             |
| created_by      | enum         | system / api            |
| created_at      | datetime     | 创建时间                    |


📌 **说明**：

* book_meta中不存在，book_item中也不会存在（需要依靠系统逻辑保证）

### 2. book_item（家庭书籍实体表）

| 字段名          | 类型          | 中文说明                     |
| ------------ | ----------- | ------------------------ |
| _id           | varchar(64) | 实体书 ID                   |
| family_id    | varchar(64) | 所属家庭                     |
| book_meta_id | varchar(64) | 关联书籍主数据                  |
| bookshelf_id | varchar(64) | 所在书架                     |
| set_index    | int         | 套装内序号（仅套装书）                    |
| inventory_status       | enum        | in_stock / off_stock |
| rfid_tid       | varchar(64)       | RFID标签的tid（对应唯一的物理标签） |
| fg_delete   | boolean | 书本逻辑删除标记，无法再次重新上架                      |
| operator   | varchar(64) | 上下架操作人                      |
| created_at   | datetime    | 入库时间                     |
| updated_at   | datetime    | 状态更新时间                     |

📌 **说明**：
* fg_delete：下架图书的彻底删除（逻辑）标志。删除前下架图书仍然可以恢复上架，但删除后图书则无法恢复上架。
* 彻底删除前必须处于 off_stock 状态。 禁止：  in_stock -> fg_delete=true

| inventory_status    | fg_delete | 含义    |
| --------- | --------- | ----- |
| in_stock  | false     | 在架    |
| off_stock | false     | 下架（可重新上架）   |
| off_stock | true      | 已彻底删除 |
| in_stock  | true      | 非法状态  |

## 四、关键索引建议

* book_meta.isbn

* book_meta.is_set

* book_item.family_id + status

* book_item.rfid_tid

## 五、未来功能扩展相关表结构（当前不实现）
### 1. user（用户表）

| 字段名        | 类型           | 中文说明                  |
| ---------- | ------------ | --------------------- |
| id         | varchar(64)  | 用户主键 ID（系统生成）         |
| openid     | varchar(64)  | 微信 openid（应用内唯一）      |
| unionid    | varchar(64)  | 微信 unionid（跨应用唯一，可为空） |
| nickname   | varchar(100) | 用户昵称                  |
| created_at | datetime     | 创建时间                  |

**唯一性规则说明：**

* openid 在同一小程序内唯一

* unionid 若存在则用于跨系统用户合并

* 系统内部统一使用 id 作为外键

**设计预留说明：**

* 本系统当前版本仅面向开发者，因此不考虑用户登录。

* 若未来支持多登录来源，可引入 user_auth 表，将 openid / unionid 迁移至该表。
